What changes — at a glance

MD requested a simpler, fewer-steps process. This mockup shows the proposed new workflow for review before any code changes.

Removed
  • • BOM Creator role & screens
  • • Separate "BOM Pending → In Progress" step
  • • Multi-branch req creation (only Alain for now)
New / Changed
  • • Sales builds BOM recipe directly (Item · Qty · Micron)
  • • Per-FG-item Printing checkbox
  • • Accountant can edit BOM if needed
  • • Two-stage MD approval (price → customer confirm → final sign)
Auto-generated
  • • Customer Code on customer create
  • • Item Code on item create
  • • Email + in-app notification at every stage

15 of 15 answered

All product questions answered. These drive the V3 spec + plan.

Codes & Formats
Q1 · Customer Code
Auto-generated CUST-0001 (zero-padded sequential, branch-agnostic)
Q2 · Item Code
FG-0001 for finished goods · RM-0001 for raw materials (separate sequences)
Q3 · Micron input
Free-text — any number allowed (no restriction list)
Q4 · BOM Recipe total per KG
No validation — sales decides (no min/max enforcement)
Currency & Pricing
Q5 · FX rate source
Existing ExchangeRates table (current V2.3 mechanism)
Q6 · Margin amount currency
Quote currency (USD / EUR) — MD thinks customer-side
Q7 · MD pricing breakdown
Full breakdown: RM / Printing / FOH / Transport / Commission shown side-by-side
Q8 · Customer-rejects flow
"Request MD to re-price" goes back to MD only (re-margin) — does NOT re-route to Accountant
Final Approval
Q9 · MD signature
Both — image upload (one-time) + text below ("Approved by [Name] on [Date]")
Q10 · Sales notes visibility
Visible to Sales + Accountant + MD (all 3 internal roles) · NOT on customer-facing PDF
⚠️ Customer email policy (added round 2)
NO external customer emails ever. All notifications (incl. signed PDF) go only to Sales + Accountant + MD. Sales downloads PDF and forwards to customer manually via own channels.
Migration & Scope
Q11 · In-flight requisitions
Hard cut-over — cancel/freeze old reqs, sales re-create needed ones in V3
Q12 · BOM Creator user accounts
Deactivate (set IsActive=false)
Q13 · Other 4 branches data
Hide entirely — only Alain visible (Dubai/Sharjah/etc customers + items hidden)
Q14 · Mobile app
Phase 1 web-only; mobile V3 deferred to Phase 2 (V2 mobile freezes)
Q15 · PWA support
Continue — web installable PWA stays (iOS/iPadOS staff workflow preserved)
