## Subagent dispatch protocol (strict tasks)

For each strict-dual-review task (F1-F4, E1, E3, E5, R1):

1. **Implementer** subagent receives the task definition + spec excerpt; produces edits + tests; reports.
2. **Spec reviewer** receives the report + spec; identifies divergence (spec said X, impl did Y).
3. If reviewer reports ≥ 1 divergence: dispatch **fix subagent** with reviewer's notes. Repeat until clean.
4. **Code reviewer** receives the (now spec-correct) impl; identifies bugs / smells / test gaps.
5. If reviewer reports issues: dispatch fix subagent. Repeat until clean.
6. Mark task done; commit.

For single-reviewer tasks (E2, E4, R2, R3, U1-U6):

1. **Implementer** subagent + a single combined spec+code reviewer.
2. One fix loop max.
3. Commit.
